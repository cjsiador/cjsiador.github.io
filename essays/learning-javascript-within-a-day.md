  I found that learning new programming languages gives me the same dopamine as staying up all night programming. The stimulation of excitement and learning new features provides a mentality of  doing certain things that other programming languages that don't have the capability of doing. I believe that JavaScript has a lot of potential for creativity, mainly as a web application.  

  Furthermore, JavaScript is a very forgiving programming language, but not as forgiving as Python. What I understand from learning JavaScript is that it’s an implicit programming language. In other words, a variable in JavaScript can be a character, integers, double, array, and float without declaring its data type. An explicit programming language such as C/C++ and Java, it requires programmers to declare a data type of that particular variable. One of the cool features that JavaScript has is the ability to mix data type within an array. With this feature, it has a lot of potential of making less lines of code. 

  However, JavaScript isn’t perfect, but it’s not necessarily a bad or a good language, instead, it depends on how it is utilized. What I understand from JavaScript is that it’s a programming language mostly for web applications and other things. It’s a language that’s friendly for those who are new to programming. The programming is very forgiving, but not as forgiving as one of the most simplistic programming languages such as Python. In fact, I had an easier time learning JavaScript than Python. 

  Nevertheless, I am still new to JavaScript, and I learned most of the basic syntax within a day. There are other features to explore that are completely new to me. I think that’s where Athletic Software Engineering comes into place. The program is a term to stimulate the intensity of going through the stress and writing efficient code. It’s supposed to train programmers to become Software Engineers. The ability to solve problems within a time span. I believe that it’s truly stressful, however,  I found it motivating because it challenges me how to solve coding problems more effectively. One of the features that Athletic Software Engineering provides is a coding problem that must be done within a specific amount of time, this is called workout of the day (WOD). WODs is something that challenges your analytical thinking on solving coding problems while under pressure and it also stimulates coding interviews. The philosophy of practicing WODs is to challenge yourself until you get a reasonable time result. I believe that it’ll work out for me because it gives me the motivation to push myself to be an actual Software Engineer. Hopefully it’ll help you too.
