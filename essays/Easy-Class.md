When I first learned about HTML and CSS, I was so amazed how websites were created. I began to experiment with other websites using the developer tool and making my own hacked site with bright green color as the background. Then, I remembered that I came across a different website. A website that’s special and unique with different formatting. Now that I understand the fundamentals of Semantic UI, I now realized that the website was using a different type of UI Frameworks. Instead of finding the hex color codes and code it into CSS or HTML, it’d be simply easier to just put a div label with a class and wrap it into something that you want to make into the color you want. Not only just the color itself, but the fundamentals of rounding buttons and etc. I don’t need to fiddle my CSS code and create a button using lines of code such as “border-radius” or “margin” while going through trials and errors.  No, instead it’ll be easier to write a class named “round green button.” Right? 

Not exactly. My mind is so used to coding basic CSS fundamentals that I found it ironic to just hard code it into my website than just using the class. Learning Semantic UI takes a lot of time and effort. There were times where I found that using Semantic UI can be very confusing and frustrating, however, it simplifies the coding and makes it easier to create content. Certainly, in my opinion, I felt that using Semantic UI is way better than coding CSS from scratch. However, there are some limitations using Semantic UI. 

For instance, in Semantic UI, there are a lot of predefined layouts such as size with a total of eight size class names that ranges from Mini (35px) to Massive (960px). If you want a specific size, it’s better to just write a CSS code and overwrite it. Another problem that I came across was the confusion of using numerical words to specify how many items in columns/rows you want and how it was padded. Hopefully, in the future, Semantic UI continues to grow with more variety of options and predefined layouts for developers to use.
