When I was in my first programming class, we were assigned to review other student’s programs. From it, I was able to learn some of their coding style and implement it into my own style as well. However, there were times that I’ve encountered some “bad” written code that became hard to trace, whereas no indentation for closed curly brackets was implemented and took me longer to debug. I realized that it’s important for others, including myself to follow a certain way of coding style so that way it enables others to understand the functionality of the program.  
	
I was recently introduced to ESLint with Intellij and applied it into my JavaScript projects. I found it very useful that it automatically showed coding style errors that made my code look nice and clean. However, I did encounter some problems with it, but it’s not as big as it sounds. Throughout coding my project, I’ve encountered a lot of red marks, indicating coding style error. This frustrates me seeing the red marks that are scattered throughout lines of code. However, despite seeing more than 20 red marks, I was able to fix it within less than a minute because it has a great functionality that automatically corrects it when you click a button. Because of those coding errors, I was able to fix my mistakes and learn from it. Now, after a week, I am able to write codes with less red marks.
